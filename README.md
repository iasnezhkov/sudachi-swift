# sudachi-swift

Swift bindings for [Sudachi](https://github.com/WorksApplications/sudachi.rs) —
the Japanese morphological analyzer from Works Applications — generated via
Mozilla [UniFFI](https://github.com/mozilla/uniffi-rs).

Targets: iOS 17+, iPadOS 17+, macOS 14+. Distributed as an SPM package with
a binary `.xcframework` target.

## Why

Swift bindings so Sudachi runs natively in iOS and macOS apps — the same way
[SudachiPy](https://github.com/WorksApplications/sudachi.rs/tree/develop/python)
provides Python bindings. Tokenization happens on-device against the bundled
dictionary; no server round-trip and no fork of sudachi.rs.

## One-time setup

```bash
# 1. Install Rust toolchain + iOS targets
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y
source "$HOME/.cargo/env"
rustup target add aarch64-apple-ios aarch64-apple-ios-sim x86_64-apple-ios aarch64-apple-darwin

# 2. Pull sudachi.rs submodule
git submodule update --init --recursive

# 3. Download the SudachiDict-core dictionary (~70 MB compressed, 207 MB on disk)
scripts/fetch-dictionary.sh

# 4. Build the xcframework (~5 minutes for first cold build)
scripts/build-ios.sh
```

## Use from Swift

```swift
import SujiSudachi

let dict = try SudachiDictionary(
    systemDictPath: "/path/to/dictionaries/system_core.dic",
    userDictPaths: [],                  // optional learner-specific dicts
    resourceDir: "/path/to/dictionaries"  // contains char.def, unk.def
)

let tokenizer = try SudachiTokenizer(dictionary: dict, mode: .c)

let morphemes = try tokenizer.tokenize(text: "毎日勉強しても上手にならない。")
for m in morphemes {
    let furigana = katakanaToHiragana(s: m.readingForm)
    print("\(m.surface) (\(furigana)) — \(m.partOfSpeech.joined(separator: "-"))")
}

// Multi-granularity: re-tokenize a substring with a different mode.
// Default mode (.c) is preserved for the next tokenize() call.
let compoundParts = try tokenizer.tokenizeWithMode(text: "国家公務員", mode: .a)
// → [国家, 公務, 員]
```

## Tests

```bash
cd swift/SujiSudachi
swift test
```

Tests run natively on macOS (your dev machine). They require the
dictionary to be present at `dictionaries/system_core.dic` — fetch via
`scripts/fetch-dictionary.sh`.

## What this package exposes

| Type | Purpose |
|---|---|
| `SudachiDictionary` | Loaded dictionary handle. Pass to tokenizer(s). |
| `SudachiTokenizer` | Stateful tokenizer with a default split mode. |
| `Morpheme` | One analyzed unit (`surface`, `readingForm`, `dictionaryForm`, `normalizedForm`, `partOfSpeech: [String]`, `synonymGroupIds: [UInt32]`, `isOov: Bool`, `wordId: UInt32`, `begin: UInt32`, `end: UInt32`). |
| `SplitMode` | `.a` (short) / `.b` (medium) / `.c` (long). |
| `SudachiError` | `.dictionaryNotFound` / `.dictionaryInvalid` / `.configInvalid` / `.tokenization` — `LocalizedError`. |
| `katakanaToHiragana(s:)` | Free helper for furigana display. |

**Indexes are codepoint-based** (`begin_c`/`end_c` in sudachi.rs terms),
not byte offsets — safe to use with Swift `String` indices.

## Performance (measured on M-series macOS)

| Op | Cold | Warm |
|---|---|---|
| `SudachiDictionary.init` | 9 ms (with 207 MB Core dict) | n/a |
| `SudachiTokenizer.init` | < 1 ms | < 1 ms |
| `tokenize(text:)` short sentence | < 1 ms | < 0.1 ms |

Cold-start dict load < 10 ms because the dict is `mmap`'d, not parsed.
